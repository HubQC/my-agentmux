## 1. Prerequisites

Before installing AgentMux, ensure you have the following dependencies:
- **Go** (1.21 or later) — for compiling the binary
- **tmux** — the terminal multiplexer AgentMux uses under the hood to isolate agent sessions.

You will also need the CLI agents you intend to run (e.g., `codex`, `claude`, `aider`, `ollama`, `openhands`).

---

## 2. Installation

Clone the repository and build the binary:

```bash
git clone https://github.com/cqi/my_agentmux.git
cd my_agentmux
go build -o agentmux .
```

Move the binary to your PATH for easiest use:
```bash
sudo mv agentmux /usr/local/bin/
```

Verify the installation:
```bash
agentmux --help
```

## 3. Configuration

AgentMux uses both global configuration and project-level configuration.

### Global Configuration (`~/.agentmux/config.yaml`)
This file is automatically generated the first time you run AgentMux. It controls global behavior:

```yaml
data_dir: /Users/username/.agentmux
default_agent_type: codex    # The agent used if -t is not specified
tmux_binary: tmux            # Path to the tmux executable
session_prefix: amux         # Prefix for tmux sessions managed by AgentMux
monitor:
  poll_interval_ms: 1000
  max_log_size_mb: 50
```

### Project Configuration (`.agentmux/config.yaml`)
Create this file in your project root to define project-specific settings, custom agents, groups, and pipelines.

```yaml
# Assign agents to groups for the TUI Tree View
groups:
  frontend:
    - react-coder
    - style-reviewer
  backend:
    - api-coder

# Define sequences of agents (pipelines)
pipelines:
  build-and-test:
    - api-coder
    - test-writer

# Define custom agents
agents:
  my-custom-coder:
    command: "codex --model gpt-4"
    env:
      DEBUG: "true"
```

## 4. Basic Operations

### Starting an Agent
Start an agent session. If the agent type isn't specified, the default (`codex`) is used.

```bash
# Start default agent (codex)
agentmux start api-coder

# Start a specific agent preset
agentmux start db-manager -t claude

# Start an agent in a specific working directory
agentmux start backend-dev -w ./backend

# Assign an agent to a group (for the tree-style dashboard)
agentmux start ui-dev -g frontend
```

### Listing Agents
View all running agent sessions.

```bash
# Flat list
agentmux list

# Tree-style grouped list
agentmux list --tree
```

### Attaching to an Agent
Attach to the isolated tmux session running the agent to interact with it directly.

```bash
agentmux attach api-coder
```
*Tip: To detach from the tmux session without stopping the agent, press `Ctrl+b` then `d`.*

### Stopping an Agent
Stop a running session.

```bash
# Stop a specific agent
agentmux stop api-coder

# Stop all agents
agentmux stop --all
```

## 5. The TUI Dashboard

AgentMux provides an interactive, real-time dashboard to monitor all your agents.

```bash
agentmux dashboard
```

### Dashboard Features:
- **Tree-Style Sidebar**: Agents are grouped by their explicit `--group`, their project config group, or their working directory.
- **Resource Monitoring**: Live CPU (%) and Memory (MB) usage for every agent.
- **Live Output**: Real-time log streaming for the selected agent. ANSI formatting and colors are perfectly preserved.
- **Side-by-Side Split Mode**: Run `agentmux dashboard --split` to launch the dashboard alongside a persistent interactive tmux terminal. Press `Enter` on an agent to instantly snap the right pane to that interactive session.

### Dashboard Controls:
- **`↑`/`k`, `↓`/`j`**: Navigate between sessions and groups
- **`Enter`**: Select a session (view logs) or toggle (collapse/expand) a group
- **`←` / `→`**: Collapse / Expand groups
- **`a`**: Attach to the highlighted agent
- **`d`**: Stop/Destroy the highlighted agent
- **`q`**: Quit the dashboard

## 6. Workflows & Plans

AgentMux supports structured implementation plans. Plans have a lifecycle: `draft` -> `approved` -> `completed`/`rejected`.

### Creating a Plan
```bash
agentmux plan create "Implement JWT Auth" -f path/to/spec.md
```

### Agent-Driven Plans
Agents can natively create plans that automatically attribute to them. (They run this command inside their tmux session).
```bash
agentmux plan create "Database Schema Update" --agent-driven
```

### Reviewing Plans
List all plans:
```bash
agentmux plan list
```

View a specific plan details:
```bash
agentmux plan show 1
```

Approve or Reject a plan:
```bash
agentmux plan approve 1
agentmux plan reject 1
```

## 7. Agent Pipelines

Pipelines allow you to run a sequence of agents automatically. When one agent finishes (exits its process), the next one begins.

Ensure you have a pipeline defined in your project's `.agentmux/config.yaml`:
```yaml
pipelines:
  feature-workflow:
    - architect-agent
    - coding-agent
    - review-agent
```

Run the pipeline:
```bash
agentmux pipeline run feature-workflow
```
AgentMux will start `architect-agent`. Once it completes successfully, it will automatically launch `coding-agent`, and so on.

## 8. Troubleshooting

**"agent binary not found on PATH"**
Ensure the CLI tool for the agent type you are trying to run is installed and available in your global PATH (e.g., `npm install -g @antigravity/codex`, `npm install -g @anthropic-ai/claude-code`).

**TUI dashboard is garbled or characters look weird**
AgentMux uses modern terminal features. Ensure your terminal supports true color and UTF-8. If logs look broken, ensure the agent CLI you are running natively outputs ANSI color codes (AgentMux preserves them automatically).

**Agents are not grouping correctly in the tree view**
Grouping priority is:
1. Explicit `--group` flag at startup.
2. Group mappings inside `.agentmux/config.yaml`.
3. Fallback: The project boundary / working directory basename.

**Checking raw AgentMux logs**
AgentMux system logs are stored at `~/.agentmux/agentmux.log`.
